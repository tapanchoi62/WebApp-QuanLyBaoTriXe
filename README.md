1. Mục tiêu

Xây dựng WebApp quản lý bảo trì cho đội xe vận tải, bao gồm theo dõi vật tư, kho và lịch sử bảo trì.

2. Chức năng chính

A. Quản lý phương tiện

- Danh sách xe: biển số, model, năm sản xuất, ghi chú.

- Xem lịch sử bảo trì của từng xe.

B. Quản lý vật tư & kho

- Danh mục vật tư: vỏ xe, bình acqui, bơm dầu, phụ tùng các loại.

- Theo dõi tồn kho theo từng kho.

- Nhập kho (IN), xuất kho (OUT), điều chỉnh tồn (ADJUST).

- Nhật ký xuất/nhập chi tiết.

C. Quản lý nhà cung cấp

- Khi nhập kho

D. Phiếu đề nghị bảo trì

- Tạo phiếu đề nghị thay thế vật tư cho xe (vỏ, bình, dầu DO…).

- Mỗi phiếu có nhiều vật tư.

- Luồng duyệt: Pending → Approved/Rejected.

- Sau khi duyệt: xuất kho tự động theo danh sách vật tư.

E. Nhật ký bảo trì

- Ghi nhận công việc bảo trì thực tế: thời gian, km, mô tả.

- Vật tư sử dụng trong bảo trì (link với kho).

F. Người dùng & phân quyền

- Tài khoản: admin, kỹ thuật, thủ kho.

- Quyền: tạo phiếu, duyệt, xuất kho, nhập kho.

3. Yêu cầu kỹ thuật

- WebApp (SPA hoặc MVC).

- Backend: .NET / NodeJS / PHP (tùy freelancer).

- Database: SQL Server.

- API JWT Authentication.

- Giao diện thân thiện, responsive.

- Có ERD + tài liệu API.

4. Kết quả mong muốn

- Hoàn thiện hệ thống CRUD + luồng duyệt + kho.

- Dashboard cơ bản.

- Source code + hướng dẫn triển khai.

- Bảo hành tối thiểu 1–3 tháng.
